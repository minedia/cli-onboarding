# Claude Code とは？

## セクション 1 ／ Claude Code ハッカソン

PCで実際にアクションを起こせる Claude を 5 分で理解する

---

## "コードを書くツール" ではありません

Claude Code は

**"コードを書くツール" ではなく、<br>"PCで実際にアクションを起こせる Claude"** です。

- ターミナルに **日本語で指示するだけ**。コマンドを覚える必要はありません
- ファイルの作成・編集・整理、Excel/CSV/PDF の操作まで実行できる
- 「相談相手」から「**実行できるアシスタント**」へ

---

## ブラウザ版 Claude との違い

| 観点 | ブラウザ版 Claude（claude.ai） | Claude Code |
|---|---|---|
| 役割 | **相談相手**（会話のみ） | **実行できるアシスタント** |
| ファイル操作 | できない | 作成・編集・整理が可能 |
| データ処理 | 限定的 | Excel / CSV / PDF を直接操作 |
| 自動化 | できない | スクリプト作成から実行まで全自動 |

---

## ポイント

- **ターミナルに日本語で指示するだけ。** コマンドを覚える必要はありません
- **ファイルサイズ制限なし・コンテキスト切れなし**、長時間タスクもOK
- **何をするか事前に確認できる** → 安心して使える

> この後のセクションで、インストールから実際の業務効率化までを体験します。

# インストール & セットアップ

## セクション 2 ／ Claude Code ハッカソン

ターミナルを開いて、Claude Code を入れてログインするまでを 1 ステップずつ

---

## まずターミナルを開こう

Claude Code を使うには「ターミナル」を開きます。
難しそうに見えますが、**文字を入力するだけの画面**です。

### ターミナルの開き方

1. キーボードの **Windows キー** を押す（左下のスタートメニューが開く）
2. そのまま「**Terminal**」と入力する
3. 「**ターミナル**」というアプリが見つかる
4. **右クリック** → 「**管理者として実行**」を選ぶ

> **「管理者として実行」を選ぶ理由：** インストール時にPCの設定を変更するため、通常より高い権限が必要です。ハッカソン中だけなので心配ありません。

---

## ターミナルとは

ターミナルが開くと、黒や青の背景に文字が点滅している画面が表示されます。

ここに文字を打ち込んで **Enter** を押すと、PC に命令を出せます。

- 表示される英語の文字に驚かなくて大丈夫
- 打ち込む内容はこの後すべて提示します
- コピー＆ペーストで進められます

---

## Claude Code をインストールする

公式ドキュメント： https://code.claude.com/docs/ja/setup

3ステップだけで完了します。

### ステップ 1：インストールコマンドを実行

下の1行をコピーして、ターミナルに貼り付けて **Enter**：

```powershell
irm https://claude.ai/install.ps1 | iex
```

> **貼り付け方：** この1行を選択して `Ctrl + C` でコピー → ターミナル上で **右クリック** するだけで貼り付けられます。貼り付けたら `Enter` キーを押してください。

---

## インストール（ステップ 1 補足）

ステップ 1 を実行すると…

- 画面に英語の文字がたくさん流れますが、**正常な動作**です
- 完了するまで **1〜3 分** ほど待ちましょう

> 流れている文字は「いまインストール中です」という進捗表示です。エラーではないので、そのまま待ってください。

---

## インストール（ステップ 2・3）

### ステップ 2：ターミナルを開き直す

インストールが完了したら、ターミナルを **× ボタンで閉じて**、もう一度「管理者として実行」で開き直します。

> **なぜ開き直すの？** インストールで追加された設定を読み込むためです。スマホのアプリ更新後に再起動するのと同じです。

### ステップ 3：動作確認

新しいターミナルで以下を入力して **Enter**：

```powershell
claude --version
```

バージョン番号（例: `2.3.0`）が表示されたら成功です！

---

## うまくいかないときは（あせらなくて大丈夫！）

インストールは PC 環境によってスムーズにいかないこともあります。

以下の「困りごと」から、自分の状況に近いものを探してください。

- 困りごと 1：「スクリプトの実行が無効です」エラー
- 困りごと 2：`claude` が「見つかりません」
- 困りごと 3：途中で止まる / 時間がかかりすぎる
- 困りごと 4：黒い画面が一瞬だけ出て消える

---

## 困りごと 1：スクリプトの実行が無効

こんなメッセージが出た場合：

```
Running scripts is disabled on this system.
```

**これは何？** → 会社のPCではセキュリティのため、外部のプログラムを簡単に動かせない設定になっていることがあります。

---

## 困りごと 1：対処法

以下の1行を貼り付けて **Enter**（実行ポリシーを変更）：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

> 「変更しますか？」と聞かれたら `Y` を入力して Enter してください。

そのあと、もう一度インストールコマンドを実行：

```powershell
irm https://claude.ai/install.ps1 | iex
```

---

## 困りごと 2：claude が見つかりません

こんなメッセージが出た場合：

```
claude : The term 'claude' is not recognized ...
```

**これは何？** → Claude Code 自体はインストールできたのに、PCが「claude はどこ？」と迷子になっている状態です。

「道案内」の設定（**PATH** といいます）が足りないだけなので、すぐ直せます。

---

## 困りごと 2：対処法（まず試す）

**まず試すこと：ターミナルを閉じて、開き直す**

これだけで直ることが多いです。

1. 今のターミナルを閉じる
2. 新しいターミナルを「**管理者として実行**」で開く
3. 次のコマンドを試す

```powershell
claude --version
```

---

## 困りごと 2：対処法（手動でPATH設定）

それでもダメなら、**道案内を手動で設定**します。
以下を **1行ずつ** 貼り付けて Enter：

```powershell
[Environment]::SetEnvironmentVariable("PATH", "$([Environment]::GetEnvironmentVariable('PATH', 'User'));$env:USERPROFILE\.local\bin", "User")
```

```powershell
$env:PATH = "$env:PATH;$env:USERPROFILE\.local\bin"
```

そのあと、もう一度確認：`claude --version`

---

## 困りごと 3：途中で止まる / 遅すぎる

**これは何？** → 会社のネットワーク（プロキシやファイアウォール）がインストールを邪魔している可能性があります。

**対処法：** 別の方法でインストールできます。以下を **1行ずつ** 貼り付けて Enter：

```powershell
winget install --id Anthropic.ClaudeCode
```

```powershell
winget install --id Git.Git -e --source winget
```

---

## 困りごと 3：winget について

> `winget` は Windows に入っている「アプリストア」のコマンド版です。Microsoft 公式なので安心して使えます。

**winget も動かない場合**

→「トラブルシューティング」を参照するか、周囲の詳しい方に相談してください。

---

## 困りごと 4：黒い画面が一瞬で消える

**これは何？** → 古いバージョンで起きることがある既知の問題です。

**対処法：** 以下を実行してアップデートしてください。

```powershell
winget upgrade Anthropic.ClaudeCode
```

---

## どれにも当てはまらないとき

エラー画面の **スクリーンショット** を撮っておくと、問題解決がスムーズです。

> 詳しい対処法 →「トラブルシューティング」を参照

---

## ログイン（認証）する

インストールができたら、Claude Code にログインします。

### ステップ 1：Claude Code を起動する

ターミナルで以下を入力して **Enter**：

```
claude
```

### ステップ 2：ログイン方法を選ぶ

画面に選択肢が表示されます。アカウントに合った方法を選んでください。

- **Claude Max / Anthropic Console** → ブラウザが自動で開く。案内に従ってログイン
- **API Key** → 取得済みのAPIキー文字列を使う

---

## ログイン：成功の確認

### ステップ 3：ログインできたか確認する

ログインが成功すると、Claude Code の入力画面が表示されます。

`>` マークが出て入力待ちになっていれば **OK** です！

> ここまで来れば認証は完了。次はログインでつまずいたときの対処を確認しておきましょう。

---

## ログインで困ったら（1）

**ブラウザが自動で開かない場合：**

ターミナルにURLが表示されているはずです。
`c` キーを押すとURLがコピーされるので、ブラウザに貼り付けてアクセスしてください。

**「OAuth error」と表示される場合：**

認証情報をリセットしてやり直します（次のスライドのコマンド）。

---

## ログインで困ったら（2）OAuth error

以下を **1行ずつ** 実行してから、再度 `claude` を起動：

```powershell
Remove-Item -Path "$env:USERPROFILE\.claude" -Recurse -Force -ErrorAction SilentlyContinue
```

```powershell
Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force -ErrorAction SilentlyContinue
```

---

## ログインで困ったら（3）最終手段

**何をやってもログインできない場合：**

APIキー方式に切り替えるのが最速です。
APIキーを用意して、以下のように入力してください。

```powershell
$env:ANTHROPIC_API_KEY = "ここにAPIキーを貼る"
claude
```

---

## 最終チェック

以下のコマンドで環境全体をチェックできます：

```powershell
claude doctor
```

- 問題がなければ **緑色** の表示が出ます
- **赤色** の項目がある場合は「トラブルシューティング」を参照

# ウォームアップ体験

## セクション 3 ／ Claude Code ハッカソン

Claude Code に最初の仕事をお願いし、「本当にこれだけで動くの？」を実感する

---

## ステップ 1：作業用フォルダを準備する

今日の作業ファイルを置くフォルダを、デスクトップに作ります。

#### 方法A：マウスで作る（おすすめ）

1. デスクトップの何もないところで **右クリック**
2. 「**新規作成**」→「**フォルダー**」を選択
3. 名前を **hackathon** にする

#### 方法B：ターミナルで作る

```powershell
mkdir ~\Desktop\hackathon
```

> `~`（チルダ）は「自分のホームフォルダ」を表す記号。上のコマンドは「デスクトップに hackathon フォルダを作って」という意味です。

---

## ステップ 2：そのフォルダで Claude Code を起動する

作ったフォルダの中で Claude Code を動かします。
以下の2行を **1行ずつ** ターミナルに貼り付けて Enter してください。

```powershell
cd ~\Desktop\hackathon
```

```powershell
claude
```

> `cd` は「移動する」コマンド。表示が `...\Desktop\hackathon>` に変わっていればOKです。少し待って `>` マークが出て入力待ちになれば準備完了。

---

## ステップ 3：「許可」の仕組みを理解する

Claude Code はあなたのパソコン上でファイル作成やコマンド実行を行います。
そのため、**何をするにもあなたの「許可」が必要** です。

#### 許可が必要な場面（例：ファイル作成）

```
╭──────────────────────────────────────────╮
│  Claude wants to write to               │
│  自己紹介.txt                            │
│                                          │
│  Allow? (y/n)                            │
╰──────────────────────────────────────────╯
```

ここがとても大事なポイントなので、先に説明します。

---

<!-- _class: compact -->

## ステップ 3：許可の確認メッセージ（つづき）

#### 許可が必要な場面（例：コマンド実行）

```
╭──────────────────────────────────────────╮
│  Claude wants to execute                 │
│  pip install openpyxl                    │
│                                          │
│  Allow? (y/n)                            │
╰──────────────────────────────────────────╯
```

こんなメッセージが出たら：

| キー | 動作 |
|------|------|
| `y` → Enter | **許可する** — Claude Code がその操作を実行します |
| `n` → Enter | **拒否する** — その操作はスキップされます |

> **ハンズオン中は基本的に `y` を押して大丈夫です。** 内容を確認してから許可してください。

---

## よく出る許可の種類

| 許可の種類 | 表示例 | 意味 |
|-----------|--------|------|
| **ファイル作成・編集** | `Write to 自己紹介.txt` | 新しいファイルを作る / 既存ファイルを書き換える |
| **コマンド実行** | `Execute: pip install ...` | ターミナルのコマンドを実行する |
| **ファイル読み取り** | （自動で許可されます） | ファイルの中身を読むだけなので許可不要 |

> ファイル読み取りは自動で許可され、確認は表示されません。

---

## 「許可が多すぎる」と思ったら

1つの指示で複数のファイルを作ったりコマンドを何回も実行したりすると、**許可の確認が何度も出る** ことがあります。
そんなときは、許可画面で **`a`（Accept All）** を選ぶとまとめて許可できます。

| キー | 動作 |
|------|------|
| `y` | この操作だけ許可（1回限り） |
| `a` | このセッション中、同じ種類の操作をすべて許可 |
| `n` | この操作を拒否 |

> **安心してください：** Claude Code は許可なしに勝手にファイルを消したり外部にデータを送ったりしません。許可は安全装置です。

---

## うっかり `n` を押してしまったら

間違えて拒否してしまっても大丈夫です。

- もう一度同じ指示を出せば、Claude Code がやり直してくれます
- 「**さっきの操作をもう一度やってください**」と日本語で伝えてもOK

> 拒否はあくまでその操作をスキップするだけ。やり直しはいつでも可能です。

---

## ステップ 4：日本語で指示を出してみよう

実際に Claude Code に仕事をお願いしてみましょう。
`>` の後ろに、以下の文章をそのまま入力して Enter を押してください。

```
この作業フォルダに自己紹介テキストを作成してください。
名前、部署、今日のハッカソンで自動化したいことを書いてください。
名前は「テスト太郎」、部署は「営業部」としてください。
```

> **ポイント:** コマンドではなく **普通の日本語** で大丈夫です。同僚にお願いするように書いてください。

---

## ステップ 5：Claude Code の動きを見てみよう

Enter を押すと、Claude Code が考えはじめます。流れは次の通りです。

#### 1. 計画を見せてくれる

いきなり作業を始めず、まず「こういうファイルを作ろうと思います」と計画を提示。

#### 2. 許可を求めてくる

ステップ3の **許可の確認** が表示される。内容を確認して `y` → Enter で許可。

#### 3. ファイルが作成される

許可すると作成完了。`hackathon` フォルダにテキストファイルが出来ています！

> AIが何をしようとしているか事前に確認でき、許可しないと実行されないので安心です。

---

## ステップ 6：会話を続けてみよう

ファイルが作成されたら、続けてこう指示してみてください。

```
今作ったファイルの内容を、もっと丁寧なビジネス文書風にリライトしてください。
箇条書きも追加してください。
```

> Claude Code は **前の会話を覚えています**。「今作ったファイル」でどれか理解してくれます。

---

## ステップ 6：さらに試してみよう

さらにこんな指示も試してみましょう。

```
このフォルダに「議事メモ.txt」を新しく作って、
今日の日付、参加者3名、決定事項3つ、次回アクション2つを
サンプルとして入れてください。
```

> 1回の指示で **複数の情報を盛り込んだファイル** を作れることが確認できます。

---

## うまくいかないときは

| こんなとき | やること |
|-----------|---------|
| 何も反応しない | 少し待つ（最初は数秒かかることがあります） |
| エラーっぽい英語が出た | `Esc` キーで一度止める。→「参加者向け Q&A」も参照 |
| 入力を間違えた | `Esc` で止めてから、もう一度入力し直せばOK |
| 元の状態に戻したい | 「今の作業を取り消してください」と日本語で入力 |

---

## 便利コマンド：操作の中断・やり直し

| 操作 | やり方 | 説明 |
|------|--------|------|
| **中断** | `Esc` キー | Claude の作業を途中で止める。会話はそのまま残る |
| **巻き戻し** | `Esc` 2回、または `/rewind` | 過去の状態に戻すメニューが開く。会話もファイルも元に戻せる |
| **取り消し** | 「今の変更を元に戻して」と入力 | Claude が直前の作業を取り消してくれる |

> `/rewind` は「さっきの方が良かった」ときに便利。すべての操作にチェックポイントが作られ、好きな時点まで戻れます。

---

## 便利コマンド：コンテキストの管理

会話が長くなると、Claude Code は過去の内容を「忘れる」ことがあります。以下で対処できます。

| コマンド | 説明 |
|----------|------|
| `/clear` | 会話をリセットして最初からやり直す。**別の話題に切り替えるときに使う** |
| `/compact` | 会話を要約して短くまとめる。会話が長くなってきたときに使う |

> **ポイント**: 1つの作業が終わったら `/clear` でリセットしてから次に入ると、Claude の精度が保たれます。

---

## 便利コマンド：セッションの管理

Claude Code を一度閉じても、前回の会話を再開できます。

| コマンド | 説明 |
|----------|------|
| `claude --continue` | 直前の会話の続きから再開する |
| `claude --resume` | 過去の会話一覧から選んで再開する |

> 途中で休憩する場合や、翌日に続きをやりたい場合に便利です。

---

## ここまでのまとめ

今の体験で、以下のことが確認できました。

- **日本語で指示するだけ** でファイルが作れる
- Claude Code は **事前に計画を見せてくれる**
- **許可しないと実行されない** ので安全（`y` / `n` / `a` で制御）
- **前の会話を覚えている** ので、続けて指示を出せる
- 間違えても **やり直し・取り消しができる**（`/rewind` や `/clear`）

> この感覚をつかめたら、次はもっと実用的なデモへ。困ったときは →「参加者向け Q&A」を参照。

# Skills で更なる効率化

## セクション 6 ／ Claude Code ハッカソン

世界中の Skills を使い、自分でも作って効率を引き上げる

---

## Skillsとは？

Skills = **「Claude Code に特定の仕事のやり方を教える説明書」** です。

一度インストールすれば何度でも再利用でき、チームで共有すれば組織全体の品質を均一化できます。

> 新入社員に「うちの会社のレポートはこう書くんだよ」とマニュアルを渡すのと同じ。Skills は **Claude への業務マニュアル** です。

---

## Skills の具体的な使い方

- インストールすると、Claude Code で **`/スキル名`** と入力するだけで呼び出せる
- 会話の内容に関連する Skills があれば、**Claude が自動で判断して使う** こともある

> 公式ドキュメント：[Claude をスキルで拡張する](https://code.claude.com/docs/ja/skills)

---

## Skills を探す — skills.sh

世界中の開発者が作った Skills が **[skills.sh](https://skills.sh/)** で公開されています。
現在 **45,000個以上** が登録され、検索して1コマンドでインストールできます。

| Skills 名 | インストール数 | 内容 |
|---|---|---|
| **find-skills** | 143K+ | 他の Skills を検索・発見 |
| **web-design-guidelines** | 78K+ | Webデザインのガイドライン |
| **frontend-design** | 48K+ | フロントエンド設計のベストプラクティス |
| **pdf** | — | PDFファイルの読み取り・処理 |
| **excel** | — | Excelファイルの操作 |

---

## インストール手順（ステップ1・2）

**ステップ 1：[skills.sh](https://skills.sh/) で気になる Skills を見つける**

- 「pdf」で検索 → PDFを扱う Skills
- 「excel」で検索 → Excel操作の Skills
- 「design」で検索 → デザイン関連の Skills

**ステップ 2：1コマンドでインストール**

```bash
npx skills add <作者名/スキル名>
# 例
npx skills add anthropic/find-skills
```

---

## インストール手順（ステップ3）

**ステップ 3：Claude Code で使ってみる**

インストールが完了したら、Claude Code を起動して `/スキル名` で呼び出せます。

```
/find-skills Excelを扱えるスキルを探して
```

> **ポイント:** Claude Code の中ではなく **ターミナル**（起動前の画面）でコマンドを実行します。起動中なら `Esc` で一度終了してからインストールしてください。

---

## 実際に試す — 体験1：Skills を検索

まず、他の Skills を検索するための Skills をインストールします。

```bash
npx skills add anthropic/find-skills
```

インストールしたら Claude Code を起動して：

```
/find-skills 業務で使えそうなスキルを探してください。
特に、PDFやExcelの操作、レポート作成に関連するものが知りたいです。
```

Claude Code が skills.sh から関連する Skills を探して提案してくれます。

---

## 実際に試す — 体験2：インストールして使う

find-skills で見つけた中から、気になるものを試します。
例：フロントエンドデザインの Skills をインストールした場合：

```bash
npx skills add ilanbiala/frontend-design
```

```
/frontend-design シンプルなタスク管理アプリのUIを作ってください
```

Skills がデザインのガイドラインを Claude に教えるため、**何も指示しなくても品質の高いデザイン** が出来上がります。

---

<!-- _class: compact -->

## 自分で Skills を作る

skills.sh に欲しいものがなければ、自分で作れます。
Skills の実体は **`SKILL.md` というテキストファイル1つ** です。

**例：週次報告書の Skills を作る** — Claude Code に頼むだけ：

```
.claude/skills/weekly-report/SKILL.md を作成してください。
以下のルールで週次報告書を生成する Skills にしてください：
- セクション構成: 1.今週の成果 2.来週の計画
  3.課題・リスク（重要度つき） 4.数値KPI（表形式）
- 文体はです・ます調 / 数値は具体的に / A4 1枚に収める
```

---

## 作った Skills を呼び出す

作成されたら、次回から `/weekly-report` で呼び出せます。

```
/weekly-report 今週は新規顧客3社獲得、売上目標達成率85%、
来週は展示会準備が主なタスクです
```

引数として状況を渡すだけで、定義したフォーマットどおりの報告書が生成されます。

---

## Skills の保存場所

| 場所 | パス | 使える範囲 |
|---|---|---|
| **個人用** | `~/.claude/skills/スキル名/SKILL.md` | 自分の全プロジェクト |
| **プロジェクト用** | `.claude/skills/スキル名/SKILL.md` | そのプロジェクトだけ |

> チームで共有したい場合は、プロジェクト用の `.claude/skills/` フォルダごと Git にコミットすれば、チーム全員が同じ Skills を使えます。

---

## カスタム Skills のアイデア

| ユースケース | Skills の内容 |
|---|---|
| 社内レポート作成 | ブランドガイドライン（色・フォント・レイアウト）を定義 |
| 社内コミュニケーション | ステータス／週次報告などのフォーマット統一 |
| 競合分析 | 決まった分析フレームワーク（SWOT等）で毎回実行 |
| 顧客対応メール | トーン・構成・必須要素を定義 |
| データ整理 | CSVの命名規則・カラム順序・バリデーションルール |

---

## まとめ

- **[skills.sh](https://skills.sh/)** で世界中の Skills を検索・インストールできる
- インストールは `npx skills add` の1コマンド
- Claude Code で `/スキル名` で呼び出すだけ
- 自分の業務に合った Skills を自作して、チームで共有もできる

> 詳しい仕様や高度な使い方は → [公式ドキュメント: Skills](https://code.claude.com/docs/ja/skills)

# Lab: 売上ダッシュボードを作る

## ハンズオン Lab ／ Claude Code ハッカソン

売上データを見える化する Web ダッシュボードを Claude Code に作ってもらう

---

## このハンズオンについて

売上データを **見える化する Web ダッシュボード** を Claude Code に作ってもらいます。
プログラミングの知識は一切不要。日本語で指示を出すだけで、プロが作ったような画面が完成します。

- **所要時間：** 約 15〜20 分
- プログラミング知識は不要、日本語の指示だけで OK

> **CSV（シーエスブイ）** とは、Excel で開けるデータファイルの形式です。カンマで区切られたテキストで、売上データや名簿などでよく使われます。

---

## このハンズオンで体験すること

1. サンプルの売上データ（CSV）を Claude Code に作ってもらう
2. そのデータから **グラフ付きの Web ダッシュボード** を自動生成する
3. ブラウザで開いて動くことを確認する
4. 日本語の指示で **デザインや機能を改善** していく

> 日本語の指示だけで、データ作成からダッシュボード化、改善まで一気通貫で体験します。

---

## Phase 1：準備

まず、作業用フォルダで Claude Code を起動します。
ウォームアップで作った `hackathon` フォルダをそのまま使います。
ターミナルに以下を **1行ずつ** 入力してください。

```powershell
cd ~\Desktop\hackathon
```

```powershell
claude
```

> `>` マークが表示されたら準備完了です。

---

<!-- _class: compact -->

## Phase 1：サンプル売上データを作ってもらう

Claude Code にサンプルデータを生成してもらいます。以下をそのまま入力してください。

```
売上データのサンプルCSVファイルを作ってください。
以下の条件でお願いします：
- ファイル名: sales_data.csv
- 200行のデータ
- 列: 日付、商品名、カテゴリ、地域、売上金額、数量
- 期間: 2024年1月〜12月の12ヶ月分
- カテゴリ: 食品、飲料、日用品、家電の4種類
- 地域: 東京、大阪、名古屋、福岡、札幌の5地域
- 売上金額はカテゴリによって現実的な範囲で変えてください
```

許可を求められたら `y` → Enter。数秒で `sales_data.csv` が `hackathon` フォルダ内に作られます。

---

## Phase 1：データ作成の確認

サンプルデータが正しく作られたかを確認します。

- 数秒で `sales_data.csv` ファイルが `hackathon` フォルダの中に作られる
- 許可を求められたら `y` を押して Enter

> **確認方法：** hackathon フォルダを開いて `sales_data.csv` ができていれば OK です。ダブルクリックすると Excel で中身を見ることもできます。

---

<!-- _class: compact -->

## Phase 2：ダッシュボードを生成する

いよいよ本番です。以下の指示を Claude Code に入力してください。

```
このフォルダにある sales_data.csv を読み込んで、
Chart.js を使ったWebダッシュボードをHTMLファイルで作ってください。

以下のグラフを含めてください：
1. 月別の売上推移（折れ線グラフ）
2. カテゴリ別の売上構成比（円グラフ）
3. 地域別の売上比較（横棒グラフ）
4. 売上トップ10の商品（棒グラフ）

見た目はモダンで綺麗なデザインにしてください。
1つのHTMLファイルに全部まとめてください。
```

---

## Phase 2：生成中に起きること

Claude Code が考え始めます。画面にコードがたくさん表示されますが、 **中身を理解する必要はありません** 。

- 許可を求められたら `y` を押して進める
- 30秒〜1分ほどで `dashboard.html` というファイルが作成される

> **Chart.js（チャートジェイエス）** とは、Web ページでグラフを描くための無料ツールです。名前だけ知っていれば大丈夫。Claude Code が全部やってくれます。

---

## Phase 3：ブラウザで確認する — 開き方

作成されたダッシュボードを見てみましょう。

1. デスクトップの `hackathon` フォルダを開く
2. `dashboard.html` ファイルを見つける
3. **ダブルクリック** する

これだけで、ブラウザ（Edge や Chrome）が自動的に開いて、 **グラフ付きのダッシュボード** が表示されます。

> **インターネット接続が必要です。** Chart.js をネットから読み込むため、オフラインでは表示できません。社内ネットワークに繋がっていれば問題ありません。

---

## Phase 3：画面に表示されるもの

開いたダッシュボードでは、以下が一目で把握できます。

- 月ごとの売上の上がり下がりが **折れ線グラフ** でわかる
- どのカテゴリが全体の何%を占めるかが **円グラフ** でわかる
- 地域ごとの売上の差が **棒グラフ** でわかる
- 売れ筋トップ10の商品が一目でわかる

> グラフを読み取るだけで、売上の傾向と構成が直感的につかめます。

---

## Phase 4：改善の指示を出してみよう

ここからが Claude Code の真骨頂です。
ダッシュボードを見て「こうしたいな」と思ったことを、 **そのまま日本語で伝える** だけで改善できます。

- 改善例 1：ダークテーマに変える
- 改善例 2：フィルター機能を追加する
- 改善例 3：集計テーブルを追加する
- 改善例 4：印刷用のレイアウト

> 完了したらブラウザで `dashboard.html` を **再読み込み**（F5キー）すると変更が反映されます。

---

```
ダッシュボードをダークテーマに変更してください。
背景を暗い色にして、文字を白系にしてください。

```

許可して完了したら、ブラウザで `dashboard.html` を **再読み込み**（F5キー）してください。
雰囲気がガラッと変わります。

## Phase 4：改善例 2 — フィルター機能を追加する

```
地域を選択するドロップダウンを追加してください。
地域を選ぶと、その地域のデータだけでグラフが更新されるようにしてください。
「全地域」という選択肢も入れてください。
```

> **ドロップダウン** とは、クリックすると選択肢が下に開く、あの選択欄のことです。

## Phase 4：改善例 3 — 集計テーブルを追加する

```
グラフの下に、月別×カテゴリ別の売上集計テーブルを追加してください。
前月比を計算して、上がっていたら緑、下がっていたら赤で表示してください。
合計行と合計列も入れてください。
```

> 前月比の色分けや合計行・列まで、文章で頼むだけで追加できます。

---

## Phase 4：改善例 4 — 印刷用のレイアウト

```
このダッシュボードを印刷したときに見やすいように、
印刷用のCSSも追加してください。A4横向きに収まるようにしてください。
```

> **CSS（シーエスエス）** とは、Web ページの見た目を整える仕組みです。ここでも名前だけわかれば大丈夫です。

## ここが Claude Code ならでは

**ブラウザ版の Claude（claude.ai）ではこの体験はできません。** その違いを整理します。

| | ブラウザ版 Claude | Claude Code |
|---|---|---|
| コードを書く | 画面に表示するだけ | **実際にファイルとして保存する** |
| コードを実行する | できない | **その場で実行・動作確認できる** |
| 複数ファイルの操作 | 1つずつコピペが必要 | **まとめて作成・編集できる** |
| データを読み込む | 手動で貼り付け | **ファイルを直接読み取れる** |
| 段階的な改善 | 毎回全体をコピペし直す | **既存ファイルをそのまま更新** |

> Claude Code は **あなたのパソコンの中で実際に動いている** ため、ファイルの作成・読み取り・書き換えが自由自在。これが「チャット」ではなく「コーディングパートナー」と呼ばれる理由です。

## 応用アイデア

今回は売上データでしたが、同じ方法で **あなたの業務データ** もダッシュボード化できます。

- **経費精算データ** → 部署別・月別の経費推移ダッシュボード
- **顧客リスト** → 地域別・業種別の顧客分布マップ
- **アンケート結果** → 回答傾向の集計・グラフ化
- **勤怠データ** → 残業時間の推移や部署別比較

> **ヒント：** Excel のデータを CSV 形式で保存して hackathon フォルダに置けば読み込ませられます。Excel の「名前を付けて保存」で「CSV（コンマ区切り）」を選ぶだけです。

## うまくいかないときは

| こんなとき | やること |
|---|---|
| CSVファイルが作られない | 「sales_data.csv を作ってください」ともう一度指示 |
| HTMLをダブルクリックしても白い画面 | インターネット接続を確認（Chart.js の読み込みに必要） |
| グラフが表示されない | ブラウザで F5 キーを押して再読み込み |
| 「Permission denied」と表示される | 許可を求める画面で `y` を押す |
| ダークテーマで文字が見えない | 「文字の色をもっと明るくしてください」と追加指示 |
| グラフのデータがおかしい | 「CSVの中身を確認してデータが正しいか検証してください」と指示 |
| 変更が反映されない | キャッシュが原因。`Ctrl + Shift + R` で強制再読み込み |
| Claude Code がフリーズした | `Ctrl + C` で中断してから `claude` で再起動 |

# Lab: 見積書ジェネレーターを作る

## ハンズオン Lab ／ Claude Code ハッカソン

日本語で指示するだけで、ブラウザで動く本格的な業務ツールを作る

---

## このLabのゴール

Claude Code に **見積書ジェネレーター** を作ってもらいます。
プログラミングの知識は一切不要。日本語で指示を出すだけです。

- **所要時間：** 約 20〜30 分
- **完成イメージ：** 会社名・商品名・数量・単価を入力すると、消費税込みの見積書がブラウザ上に表示
- **さらに：** PDF出力や履歴管理までできる Web アプリケーション

> 最初は30秒で作った単機能ツールが、会話を重ねるだけで本格的な業務アプリへ進化します。

---

## Phase 1：準備

ウォームアップ体験で作った `hackathon` フォルダを使います。
ターミナルに以下を **1行ずつ** 貼り付けて Enter してください。

```powershell
cd ~\Desktop\hackathon
```

```powershell
claude
```

`>` マークが出て入力待ちになれば準備完了です。

> **ヒント：** `hackathon` フォルダがまだない場合は、デスクトップで右クリック → 新規作成 → フォルダー で作ってください。

---

## Phase 2：基本の見積書ツールを作成する

さっそく Claude Code に見積書ツールを作ってもらいましょう。
以下のプロンプトをコピーして、`>` の後ろに貼り付けて Enter を押します。

```
会社名、商品名、数量、単価を入力すると見積書が表示されるWebツールを作ってください。
```

## うまくいかないときは

| こんなとき | 原因 | やること |
|-----------|------|---------|

# Lab: Web情報を収集・分析する

## ハンズオン Lab ／ Claude Code ハッカソン

外部データを取得・加工・レポート化する一連の作業を、日本語の指示だけで自動化

---

## この Lab でやること

外部のデータを取ってきて、加工して、レポートにまとめる。
この一連の作業を **日本語の指示だけ** で自動化する体験です。

- 今回のテーマは **為替レートの分析レポート作成**
- プログラミングの知識は一切不要
- Claude Code に「やりたいこと」を伝えるだけで進められる

> Phase 1〜6 を順に進めます。Phase 5 以降は余裕があればチャレンジ。

---

## Phase 1：準備

ウォームアップで作った `hackathon` フォルダで Claude Code を起動します。
ターミナルに以下を 1 行ずつ貼り付けて Enter してください。

```powershell
cd ~\Desktop\hackathon
```

```powershell
claude
```

`>` マークが出たら準備完了です。

> **ポイント:** 今回は Claude Code がインターネットにアクセスしてデータを取得します。社内ネットワークでWeb閲覧が制限されている場合は、ネットワーク管理者に確認してください。

---

## Phase 2：公開APIからデータを取得する

まずは為替レートのデータを集めるところから。
以下をそのまま Claude Code に入力してください。

```
過去30日間の為替レート（USD/JPY と EUR/JPY）を取得して、
CSVファイルに保存してください。
無料で認証不要の公開APIを使ってください。
```

完了すると `hackathon` フォルダに CSV ファイルが出来上がります。

> **API とは？** アプリ同士がデータをやり取りする窓口。「為替データください」とリクエストすると決まった形式で返してくれるWebサービス。認証不要の無料APIを使うのでアカウント登録は不要です。

---

## Phase 2：Claude Code がやってくれること

1. 無料で使える為替データのAPI（例：frankfurter.app など）を自動で選定
2. APIにアクセスするスクリプト（Python等）を作成
3. スクリプトを実行してデータを取得
4. `exchange_rates.csv` のようなファイルに保存

画面に「ファイルを作成してもいいですか？」「コマンドを実行してもいいですか？」と確認が出ます。
内容を見て問題なさそうなら `y` で許可してください。

---

## Phase 3：データを加工・分析する

CSV ファイルができたら、次はそのデータを分析します。
以下をまとめて入力してください。

```
取得した為替データから以下の分析を行ってください：

- 30日間の推移グラフ（HTMLファイルで作成）
- 平均・最大・最小・標準偏差の統計サマリー
- 直近1週間のトレンド（上昇/下降/横ばい）の判定
- 来週の簡易予測（移動平均ベース）
```

> **移動平均とは？** 直近数日間のデータを平均して傾向をつかむ方法。株価や為替の分析で広く使われ、Claude Code がこの計算を自動で行ってくれます。

---

## Phase 3：Claude Code がやってくれること

1. CSV データを読み込んで統計計算を実行
2. グラフ付きの HTML ファイルを生成（ブラウザで開けます）
3. トレンド判定ロジックを作って上昇/下降/横ばいを判定
4. 移動平均から来週の簡易予測値を算出

生成された HTML ファイルをダブルクリックすると、ブラウザで為替の推移グラフが表示されます。

---

## Phase 4：レポートに統合する

分析結果がそろったら、提出用のレポートにまとめます。

```
分析結果をまとめて、経理部に提出できる為替レポート（Excel）を作成してください。
グラフ付きで、A4印刷を想定したレイアウトにしてください。
以下のシートを含めてください：
- サマリー（概要・結論）
- 日次データ（CSV の内容をテーブルで）
- グラフ（推移チャート）
- 統計分析（平均・最大・最小・トレンドなど）
```

---

## Phase 4：Claude Code がやってくれること

1. Excel ファイル生成用のスクリプトを自動作成
2. 必要なライブラリ（openpyxl 等）を自動インストール
3. 複数シート構成の Excel ファイルを生成
4. グラフ・罫線・ヘッダーなどの書式設定も自動対応

完成した Excel ファイルをダブルクリックして開いてみてください。
きれいに整形されたレポートが出来上がっているはずです。

> **ライブラリの自動インストール:** 必要なツールが入っていなければ Claude Code が自動で提案。「○○をインストールしていいですか？」と聞かれたら `y` で許可してください。

---

## Phase 5：発展 — 複数の情報源を統合する

ここからは応用編です。余裕があればチャレンジしてみましょう。

### 5-1. 別のデータを追加する

```
為替との相関を分析に追加してください。

```

Claude Code が自動で日経平均のデータ取得先を探し、為替データと組み合わせた分析を実行。
相関係数や散布図がレポートに追加されます。

---

## Phase 5-2：全工程を1コマンドにまとめる

```
このレポート生成を1コマンドで実行できるスクリプトにまとめてください。
来月もう一度実行するだけで最新データのレポートが作られるようにしてください。
実行方法は日本語のREADMEに書いてください。
```

Claude Code が、今まで行ったすべての工程を1つのスクリプトにまとめてくれます。
次回からはそのスクリプトを実行するだけで、最新データの取得からレポート作成まで全自動で行えます。

---

## ここがClaude Codeならでは

> **ブラウザ版の Claude との違い**
>
> ブラウザで使う Claude（claude.ai）は、インターネットへのアクセスが限定的で、ファイルの保存もできません。一方 **Claude Code** は：
> - 実際に **HTTP リクエストを送信** してインターネット上のデータを取得できる
> - 取得したデータを **ファイルとして保存** できる
> - Python スクリプトを書いて **その場で実行** できる
> - Excel や HTML など **実務で使える形式のファイル** を直接生成できる
>
> 「調べる」「加工する」「ファイルを作る」をすべて自動実行できるのが最大の特長です。

<!-- _class: compact -->

## Q1. Claude Code って何ですか？普通の Claude と何が違うの？

### Q. 普通の Claude（claude.ai）との違いは？

**A.** ブラウザ版の Claude は「チャットで相談する相手」。一方 Claude Code は「あなたの PC で実際に作業してくれるアシスタント」です。

- ファイルの整理、Excel の集約、レポート作成などを **日本語で指示するだけ** で実行
- 名前に「Code」とあるが、**プログラミング専用ツールではない**

> 詳しくは → 「Claude Code とは？」を参照

---

## Q2・Q3. 前提知識とターミナル

### Q2. プログラミングの知識は必要ですか？

**A.** 不要です。ターミナル（黒い画面）に日本語で「〇〇してください」と入力するだけ。コマンドや言語を覚える必要はありません。

### Q3. 「ターミナル」って何ですか？

**A.** PC に文字で指示を出すための画面です。Windows では「Terminal」アプリまたは「PowerShell」。スタートメニューで「Terminal」と検索すると出てきます。

---

<!-- _class: compact -->

## Q4. 自分の PC のデータが外部に送られたりしませんか？

### Q. データの送信範囲は？

**A.** Claude Code はあなたの PC 上で動作します。ただし次のものは Anthropic の API サーバーに送信されます。

- AI に指示を出す際の **プロンプト（入力テキスト）**
- AI が読む **ファイルの内容**

会社の機密データを扱う場合は社内のセキュリティポリシーに従ってください。本日は **サンプルデータ** での作業を推奨します。

---

## Q5. 料金はどうなりますか？

### Q. 費用はかかる？

**A.** 本日のハッカソンでは **用意されたアカウント** を使用するため、個人の費用はかかりません。

今後ご自身で使う場合は、次のいずれかを選べます。

- **Claude Max**（月額 $20〜$300 の定額制）
- **API 従量課金**

## Q6・Q7. 所要時間と管理者権限

### Q6. インストールにどのくらい時間がかかりますか？

**A.** 順調なら 5 分程度。環境によっては 10〜15 分かかる場合があります。うまくいかないときは → 「トラブルシューティング」を参照。

> インストール手順 → 「インストール & セットアップ」を参照

### Q7. 管理者権限は必要ですか？

**A.** インストーラー自体は権限なしでも動きますが、管理者として実行した方がトラブルが少ないです。PowerShell を「管理者として実行」で開いてください。

---

<!-- _class: compact -->

## Q8. 社内のセキュリティソフトやプロキシが邪魔しませんか？

### Q. 企業ネットワークでの注意点は？

**A.** プロキシやファイアウォールが影響する場合があります。以下の URL へのアクセスが必要です。

- `claude.ai`（インストール・認証）
- `api.anthropic.com`（API 通信）
- `storage.googleapis.com`（インストーラーのダウンロード）

アクセスできない場合はネットワーク管理者に確認してください。

---

<!-- _class: compact -->

## Q9. 「成功」と出たのに `claude` コマンドが使えません

### Q. インストール成功後にコマンドが見つからない

**A.** Windows 版で最もよくあるトラブルです。以下を順番に試してください。

1. **まずターミナルを完全に閉じて、新しく開き直す**（これだけで直ることが多い）
2. それでもダメなら、「インストール & セットアップ」の「困りごと 2」にある PATH 設定を試す

> 詳細 → 「トラブルシューティング」を参照

---

<!-- _class: compact -->

## Q10. 認証（ログイン）の画面が出てきたらどうすれば？

### Q. 認証方法の選択画面が出たら？

**A.** `claude` と入力すると認証方法の選択画面が出ます。

- ブラウザが自動で開くので、**指示されたアカウント** でログイン
- 自動で開かない場合は `c` キーを押すと URL がコピーされるので、ブラウザに貼り付けてアクセス

## Q11・Q12. 日本語指示と安全性

### Q11. 日本語で指示を出しても大丈夫？

**A.** はい、自然な日本語で OK です。「このフォルダのファイルを整理して」「この表から売上トップ 10 を出して」など、普段同僚に頼むような言い方で指示してください。

### Q12. 勝手にファイルを消したり壊したりしませんか？

**A.** 「Plan Mode」で AI が何をするか事前に計画を見せてくれます。確認してから OK/NG を選べます。念のため **大切なファイルは作業前にコピー** を取っておくのがおすすめです。

---

## Q13・Q14. 中断と取り消し

### Q13. 途中で止めたい場合は？

**A.** `Esc` キーで Claude Code の動作を中断できます。`Ctrl + C` でも停止できます。

### Q14. 間違った指示を出してしまった場合は？

**A.** いくつかの方法があります。

- **日本語で依頼**: 「今の作業をやめて、元に戻してください」
- **`/rewind`**: 過去の任意の時点まで会話とファイルの両方を巻き戻せる（`Esc` 2 回でも同じ）
- **Plan Mode**: 事前確認の習慣で安全に

---

<!-- _class: compact -->

## Q14-2. 会話が長くなって反応がおかしくなった気がします

### Q. 会話が長くなり精度が下がったとき

**A.** Claude Code は会話が長くなると精度が下がることがあります。以下を試してください。

- **`/clear`**: 会話をリセットして最初からやり直す。別の話題に切り替えるときは必ず使う
- **`/compact`**: 会話内容を要約して短くまとめる。同じ話題を続けたいが会話が長い場合に使う

> 1 つの作業が終わったら `/clear` してから次の作業に入るのがベストプラクティスです。

---

## Q14-3・Q15. 再開とファイル利用

### Q14-3. Claude Code を閉じてしまいました。続きはできますか？

**A.** はい。`claude --continue` で直前の会話の続きから再開できます。複数の会話がある場合は `claude --resume` で一覧から選べます。

### Q15. 自分の Excel や業務ファイルを使っても大丈夫？

**A.** 本日はサンプルデータ推奨。業務ファイルを使う場合は、機密・個人情報を避け、必ずバックアップを取り、API にファイル内容が送信されることを理解した上で使用してください。

---

## Q16. 画像を貼り付けたいのですが？

### Q. クリップボードの画像を貼れる？

**A.** 現在の Windows 版では、`Win + Shift + S` でキャプチャした画像の `Ctrl + V` ペーストには対応していません。

画像を **ファイルとして保存** してから、Claude Code のウィンドウに **ドラッグ & ドロップ** してください。

---

## Q17. Skills って何ですか？

### Q. Skills の役割は？

**A.** 「Claude に仕事のやり方を教えるマニュアル」です。

たとえば「週次報告書はこのフォーマットで書いて」というルールを一度 Skill として保存しておけば、毎回同じ品質のアウトプットが得られます。

> 詳しくは → 「Skills で更なる効率化」を参照

---

## Q18. Skills は自分で作れますか？

### Q. 自作はできる？

**A.** はい。テキストファイル（SKILL.md）を 1 つ作るだけです。プログラミングは不要です。

また、[skills.sh](https://skills.sh/) で他の人が作った Skills を検索・インストールすることもできます。

> 作成手順 → 「Skills ハンズオン」を参照

---

<!-- _class: compact -->

## Q19. 作った Skills はチームで共有できますか？

### Q. チーム配布と互換性は？

**A.** はい。Skills はファイルなので、共有フォルダや Git などでチームメンバーに配布できます。

Skills は [Agent Skills](https://agentskills.io) オープンスタンダードに準拠しており、Claude Code 以外の AI ツール（GitHub Copilot、Cursor など）でも使えます。

## 1. Claude in Excel

**ExcelにAIが住みつく**

Excel のアドインとして動く公式ツール

---

## Claude in Excel — どんな機能？

**Claude in Excel** は、Excel のアドイン（拡張機能）として動く公式ツールです。
Excel を開いたまま、スプレッドシートの中で直接 Claude に質問や作業を頼めます。

> 公式ページ：[Claude in Excel](https://claude.com/ja-jp/claude-in-excel)

---

## Claude in Excel — できること

| 機能 | 説明 |
|------|------|
| **数式の解読** | 複雑な入れ子の数式を「何をしているか」と聞くだけで日本語で解説。セル参照付きで回答 |
| **エラー修正** | `#REF!` `#VALUE!` 循環参照などの原因を特定し、修正方法を提示 |
| **シナリオテスト** | 「売上が10%減ったら利益は？」を依存関係を維持したままシミュレーション |
| **モデル構築** | 「3年間の収支予測モデルを作って」でゼロからシートを構築 |

---

## Claude in Excel — 使い方

公式ガイド：https://support.claude.com/ja/articles/12650343-claude-in-excel

1. Excel のアドインストアから **Claude in Excel** をインストール
2. Excel 上でショートカットキーを押してチャット画面を開く
   - **Mac:** `Control + Option + C`
   - **Windows:** `Control + Alt + C`
3. 日本語で質問や指示を入力するだけ

---

| やりたいこと | おすすめツール |
|-------------|--------------|
| 新しいExcelレポートをゼロから自動生成 | **Claude Code** |
| 既存のExcelファイルの数式を理解・修正 | **Claude in Excel** |
| 複数CSVからグラフ付きレポート作成 | **Claude Code** |

> **ポイント:** Claude Code は「ファイルを作る・加工する」、Claude in Excel は「今開いているシートを理解・改善する」のが得意。目的に応じて使い分けを。

## 2. Claude × PowerPoint

**プレゼン資料をAIと作る**

スライドを効率的に作成・編集する2つの方法

---

## Claude × PowerPoint — 2つの方法

Claude を使って PowerPoint のスライドを効率的に作成・編集できます。
方法は大きく2つあります。

- **方法A：** Claude Desktop でスライドを **自動生成** する
- **方法B：** Claude in PowerPoint アドインで既存スライドを **編集** する

> 参考記事：[Claude Opus 4.6でパワポ作成を支援してもらった](https://dev.classmethod.jp/articles/claude-opus-4-6-powerpoint-creation-support/)

---

## 方法A：Desktop でスライドを自動生成

Claude Desktop（ブラウザ版 claude.ai でも可）に指示を出して、PowerPoint ファイル（.pptx）をまるごと生成してもらう方法です。

### 手順

1. Claude Desktop または claude.ai を開く
2. 企業テンプレート（.pptx）があればアップロードする
3. 内容を指示して生成（次スライドにプロンプト例）

---

## 方法A：プロンプト例

```
添付したテンプレートを使って、以下の内容で10枚のプレゼン資料を作ってください。

テーマ：来期の営業戦略
構成：
- 表紙
- 今期の実績サマリー
- 課題と機会の分析
- 来期の重点施策3つ（各1枚）
- スケジュール
- 予算
- まとめ
```

---

## 方法A：ポイント

- テンプレートを渡すと **企業フォーマットに合わせて** 生成してくれる
- テンプレートなしでもシンプルなデザインで生成可能
- **10〜15分程度** でスライド一式が完成

---

## 方法B：アドインで既存スライドを編集

PowerPoint のアドインとして Claude を使い、開いているスライドをリアルタイムで編集する方法です。

### できること

| 機能 | 実用度 | 説明 |
|------|--------|------|
| **テキスト修正** | 高い | 「箇条書きに変更」「もっと簡潔に」などに正確に対応 |
| **トークスクリプト生成** | 中程度 | 全スライドの発表原稿を自動生成。肉付けは必要 |
| **図解の生成** | 発展途上 | 図形での図解は可能。複雑なものは別ツール併用推奨 |

---

## 方法B：使い方とプロンプト例

1. PowerPoint のアドインストアから **Claude** をインストール
2. スライド編集画面の右側にチャットウィンドウが開く
3. 編集したいスライドを選んだ状態で、日本語で指示

```
このスライドの文章を箇条書き3つに整理してください
```

```
このスライド全体のデザインをもう少しすっきりさせてください
```

## 成功のための Tips

1. **事前にサンプルデータを用意** — 実業務データの方がWow体験が大きいが、ダミーデータも準備
2. **インストールは可能なら事前に済ませる** — 当日のトラブルで時間を食われないよう
3. **小さな成功体験を最初に** — ファイル1個の処理から始め、成功したら規模を拡大
4. **プロンプトは日本語でOK** — 自然な日本語で指示を出せることを強調
5. **バックアップの声掛け** — 重要なファイルは必ずコピーを取ってから作業

---

## Tips ／ インストールフェーズ

最も時間が読めないフェーズ。段階的に進めて滞留を防ぐ。

- **全員同時にやらない。** 3-4人ずつ段階的に進め、先に完了した人にはウォームアップ課題を渡す
- **最初にクイックコマンド集（→「クイックコマンド集」を参照）を画面共有／印刷して配る。** 参加者が自分でコピペできる状態にする
- **15分でインストールできなかった人は一旦APIキー方式に切り替える。** ハンズオンの時間を確保するため
- **ペアを組ませる。** 1人がうまくいかなくても、隣の人の画面を見ながら学べる

---

## Tips ／ ハンズオンフェーズ

- **「まず1ファイルでテスト、うまくいったら全体に適用」** を繰り返し伝える
- **Plan Mode をオンにすることを最初に全員に指示する**
- バックアップを取らせてから作業させる
- エラーが出たら「エラーメッセージをそのまま Claude Code に貼り付けて相談して」と案内（Claude Code 自身がエラーを診断できる）

> 困ったときの当日対応は →「トラブルシューティング」を参照。

# トラブルシューティング

## 運営向け ／ Claude Code ハッカソン

---

## 緊急時の対応フロー（インストール系）

<!-- _class: compact -->

```
参加者が困っている
    |
    +-- インストール系 → まず「ターミナルを新しく開き直して」と案内
    |   |
    |   +-- 直った → 続行
    |   +-- 直らない → PATHを手動追加（3分で完了）
    |       |
    |       +-- 直った → 続行
    |       +-- 直らない → winget方式に切り替え（5分で完了）
    |           |
    |           +-- 直った → 続行
    |           +-- 直らない → APIキー + 手動ダウンロード（最終手段）
```

## 緊急時の対応フロー（その他系統）

```
参加者が困っている
    |
    +-- 認証系 → APIキー方式に切り替え（最速の解決策）
    |
    +-- ネットワーク系 → スマホのテザリングに切り替え
    |   （社内プロキシを回避）

```

## トラブル1: `claude` コマンドが認識されない（最頻出）

**症状:**

```
claude : The term 'claude' is not recognized as the name of a cmdlet,
function, script file, or operable program.
```

**原因:** インストーラーが PATH 環境変数を自動設定しない既知の問題

> 最も頻出するトラブル。Step 1 から順に切り分けていきます。

---

## トラブル1: Step 1 — ターミナルの再起動

```
→ PowerShellウィンドウを完全に閉じる
→ 新しいPowerShellウィンドウを「管理者として実行」で開く
→ claude --version を実行
```

これだけで直るケースが多い。直らなければ Step 2 へ。

---

## トラブル1: Step 2 — claude.exe の存在確認

```powershell
# claude.exe がどこにあるか確認
Test-Path "$env:USERPROFILE\.local\bin\claude.exe"
```

- 結果が **True** → PATH の問題。**Step 3** へ
- 結果が **False** → バイナリが作られていない。**Step 4** へ

---

## トラブル1: Step 3 — PATH を手動で追加

```powershell
# 恒久的にPATHに追加（ユーザー環境変数）
[Environment]::SetEnvironmentVariable(
  "PATH",
  "$([Environment]::GetEnvironmentVariable('PATH', 'User'));$env:USERPROFILE\.local\bin",
  "User"
)

# 現在のセッションにも即座に反映
$env:PATH = "$env:PATH;$env:USERPROFILE\.local\bin"

# 確認
claude --version
```

> コピペ用 →「クイックコマンド集」を参照

---

## トラブル1: Step 4 — wingetで再インストール

バイナリが存在しない場合（Step 2 が False）。

```powershell
# winget でインストール（ネイティブインストーラーの代替）
winget install --id Anthropic.ClaudeCode

# Git も必要
winget install --id Git.Git -e --source winget

# ターミナルを再起動して確認
claude --version
```

---

## トラブル1: Step 5 — 直接ダウンロード（最終手段）

それでもダメなら手動でバイナリをダウンロード。

```powershell
$GCS_BUCKET = "https://storage.googleapis.com/claude-code-dist-86c565f3-f756-42ad-8dfa-d59b1c096819/claude-code-releases"
$version = Invoke-RestMethod -Uri "$GCS_BUCKET/stable"
$downloadPath = "$env:USERPROFILE\Downloads\claude-installer.exe"
Invoke-WebRequest -Uri "$GCS_BUCKET/$version/win32-x64/claude.exe" -OutFile $downloadPath

# インストール実行
& $downloadPath install stable

# PATHを追加（Step 3参照）
```

## トラブル2: ExecutionPolicy エラー

**症状:**

```
irm : Running scripts is disabled on this system.
File ... cannot be loaded because running scripts is disabled.
```

**原因:** PowerShellのスクリプト実行ポリシーが Restricted に設定されている

---

## トラブル2: 対処

```powershell
# 現在のポリシー確認
Get-ExecutionPolicy

# ユーザースコープで変更（管理者権限不要）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 確認（→ RemoteSigned と表示されればOK）
Get-ExecutionPolicy

# 再度インストールコマンドを実行
irm https://claude.ai/install.ps1 | iex
```

> **注意:** 企業のグループポリシーで制限されている場合、ユーザースコープでも変更できないことがある。その場合は winget 方式に切り替える。

## トラブル3: Git Bash で実行してしまった

**症状:**

```
Error: Raw mode is not supported on the current process.stdin
```

**原因:** Git Bash は Claude Code が必要とする TTY 機能をサポートしていない

**対処:**

```
→ Git Bash を閉じる
→ Windows PowerShell または Windows Terminal を使う
→ スタートメニュー → 「Terminal」で検索 → PowerShell タブで開く
```

---

## トラブル3: 参加者への説明

「黒い画面」は何種類かあります。

- Claude Code では **PowerShell（青い背景のもの）** か **Windows Terminal** を使う
- **Git Bash（MINGW64 と表示されるもの）は使えません**

> 画面上部やプロンプトの表示で、どのターミナルかを見分けられます。

## トラブル4: OAuth 認証が完了しない

### 症状A: ブラウザが自動で開かない

```
対処:
→ ターミナルに表示されたURLを `c` キーを押してコピー
→ ブラウザに手動で貼り付けてアクセス
→ 認証コードが表示されたらターミナルに貼り付け
```

---

## トラブル4: 症状B — "OAuth error" が表示される

```
対処:
1. 認証情報をクリア
   Remove-Item -Path "$env:USERPROFILE\.claude" -Recurse -Force
   Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force

2. claude を再起動して再度ログイン

3. それでもダメなら APIキー方式に切り替え
   $env:ANTHROPIC_API_KEY = "sk-ant-xxxxx"
   claude
```

---

## トラブル4: 症状C — "OAuth account information not found"

`OAuth account information not found in config` エラー。

```
対処:
1. ブラウザのキャッシュ・Cookieをクリア
2. ブラウザのシークレットモードで認証を試す
3. ~/.claude フォルダを削除して再ログイン
4. 最終手段: APIキー方式に切り替え
```

---

## トラブル4: 当日のTip

OAuthトラブルが多発しそうな場合、最初から **APIキー方式** を全員に配布するのが最も安全。

```powershell
# APIキーを環境変数に設定
$env:ANTHROPIC_API_KEY = "sk-ant-api03-xxxxxxxx"
claude
```

## トラブル5: プロキシ / ファイアウォール問題

**症状:**

```
Request timed out. Check your internet connection and proxy settings
```

**対処:**

```powershell

```

## トラブル8: よくある出力と対処

<!-- _class: compact -->

| 出力 | 対処 |
|------|------|

## トラブル9: Claude Codeが途中でフリーズ・応答しない

**対処:**

```
1. Esc キーを押す（操作の中断）
2. Ctrl + C を押す（強制停止）
3. それでも止まらない場合 → ターミナルウィンドウを閉じる
4. 新しいターミナルを開いて claude を再実行
5. 長時間タスクの場合 → /compact でコンテキストを圧縮してからリトライ
```

## トラブル10: コンソールウィンドウが一瞬点滅して消える

**症状:** claude 実行時にコマンドプロンプトの窓が一瞬開いて閉じる

**原因:** v2.1.27 で修正された既知の問題だが、古いバージョンでは発生する

**対処:**

```powershell
claude --version

# 古い場合はアップデート

# または再インストール
irm https://claude.ai/install.ps1 | iex
```

# クイックコマンド集

## 運営向け ／ Claude Code ハッカソン

参加者のPCで素早くコピペできるよう主要コマンドをまとめています。画面共有または印刷して配布してください。

---

## インストール ／ PATH修正

### インストール

```powershell
irm https://claude.ai/install.ps1 | iex
```

### PATH修正（インストール後に claude が見つからない場合）

```powershell
[Environment]::SetEnvironmentVariable("PATH", "$([Environment]::GetEnvironmentVariable('PATH', 'User'));$env:USERPROFILE\.local\bin", "User")
$env:PATH = "$env:PATH;$env:USERPROFILE\.local\bin"
```

---

## ExecutionPolicy修正 ／ 代替インストール

### ExecutionPolicy修正

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 代替インストール（winget）

```powershell
winget install --id Anthropic.ClaudeCode
winget install --id Git.Git -e --source winget
```

---

## APIキー認証 ／ 環境診断

### APIキーでの認証（OAuth不要）

```powershell
$env:ANTHROPIC_API_KEY = "ここにAPIキーを貼る"
claude
```

### 環境診断

```powershell
claude doctor
claude --version
```

---

## 認証情報リセット ／ プロキシ設定

### 認証情報リセット

```powershell
Remove-Item -Path "$env:USERPROFILE\.claude" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "$env:USERPROFILE\.claude.json" -Force -ErrorAction SilentlyContinue
```

### プロキシ設定

```powershell
$env:HTTPS_PROXY = "http://proxy-server:port"
$env:HTTP_PROXY = "http://proxy-server:port"
```

---

## 作業フォルダ作成 ／ セッション再開

### 作業フォルダ作成 & 移動

```powershell
mkdir "$env:USERPROFILE\Desktop\hackathon" -Force
cd "$env:USERPROFILE\Desktop\hackathon"
claude
```

### セッション再開

```powershell
claude --continue       # 直前の会話を再開
claude --resume         # 過去の会話一覧から選んで再開
```

## Claude Code 操作コマンド（セッション内）

Claude Code 起動中に `>` プロンプトで使える便利コマンドです。

```
Esc           … 実行中の作業を中断
Esc + Esc     … 巻き戻しメニューを開く（/rewind と同じ）
/clear        … 会話をリセット（別の作業に切り替える前に）
/compact      … 会話を要約して圧縮（会話が長くなったときに）
/rewind       … チェックポイントに巻き戻す
/init         … CLAUDE.md（プロジェクト設定）を自動生成
/help         … ヘルプ表示
```
